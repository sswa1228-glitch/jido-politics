🌿 브랜치 전략

main

항상 배포 가능한 상태. PR merge만 허용. 태그 = 버전 릴리즈.

develop

개발 통합 브랜치. feature/* 브랜치가 여기로 merge. CI 통과 후 main에 PR.

feature/*, fix/*

기능 단위 브랜치. 예: feature/comment-system fix/map-loader

📝 커밋 컨벤션 (Conventional Commits)

형식: type(scope): 설명

feat새 기능 추가 — feat(map): 드릴다운 애니메이션 추가
fix버그 수정 — fix(loader): map-loader CSS display:none 기본값 수정
docs문서 변경 — docs: roadmap_v2.1 업데이트
refactor기능 변경 없는 코드 개선
chore빌드/설정 변경 — chore: vite 설정 추가
styleCSS·포맷 변경 (기능 무관)

⚡ Vercel 배포 (대안, 더 빠름) 권장

Vercel은 GitHub 연동 자동 배포 + 커스텀 도메인 + HTTPS 무료.

# Vercel CLI로 배포
npm i -g vercel
cd prototype/
vercel  # 첫 실행 시 GitHub 연동 + 자동 설정

이후 git push만 하면 자동으로 프리뷰 URL 생성 + main merge 시 프로덕션 배포.
