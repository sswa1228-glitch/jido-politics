⚡ 성능 전략

districts.stat_* 필드는 실시간 집계 대신 캐시 컬럼으로 운영 — 게시글 등록/삭제 시 트리거로 업데이트
posts에 복합 인덱스 (district_id, political_lean, status, created_at) — 지역구별 탭 조회 최적화
news_articles.url_hash UNIQUE 인덱스로 크롤링 중복 방지
posts.deleted_at NULL 체크로 soft delete — 복구 가능, 통계에서 제외

🛡️ 어뷰징 흐름

게시글 등록 → AI 1차 판단 (ai_score 저장) → 기준 미달 시 status = pending
신고 5회 누적 (report_count ≥ 5) → 자동 관리자 큐 적재
관리자 처리 시 moderation_log에 전체 이력 기록 — source 필드로 AI/신고/수동 구분
반복 어뷰저 → users.warning_count 누적 → 3회 시 status = banned 자동 전환

📍 위치 인증 로직

GPS 좌표 수신 → PostGIS 또는 Turf.js로 districts 경계와 교차 판정
인증 성공 시 users.district_id 업데이트 + location_verified_at 기록
인증 유효기간 30일 — 만료 시 재인증 요구 (앱 재실행 시 체크)
게시글 작성 시 posts.location_verified = true 스냅샷 — 추후 분쟁 대비

🗞️ 뉴스 크롤링 설계

크롤러 (Python) → 지역구명 키워드 검색 → 기사 수집
Claude API → 기사 요약 (summary) + 지역 연관도 점수 (relevance_score) + 정치 톤 분류
is_national = true 상위 5건 → 상단 티커 표시 / 나머지 → 지역구 말풍선
크롤링 주기: 전국 뉴스 30분 / 지역 뉴스 2시간 단위
